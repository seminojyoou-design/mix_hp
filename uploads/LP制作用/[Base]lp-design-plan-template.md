### トリミング要否

- [ ] 不要物（スマホUI、白線等）が写り込んでいる画像はあるか

### 画像ルール

- 元画像は加工しない（リサイズ・圧縮しない）

- 画像の実サイズを超える表示はしない → `max-width` で制限
- 超えた分は背景色（--warm-white等）で余白にする

## 2. レイアウト設計

### コンテンツ幅の決定

**画像の最小幅に基づく max-width：** ○○px
**PC時のコンテンツ幅：** ○○%（参考：eLan=76%）
**余白の処理：** 左右を背景色で埋める

### 各セクション構成

| セクション | レイアウト型 | 画像配置 | 画像サイズ制御 |
|-----------|------------|---------|-------------|
| ヒーロー | フルスクリーン / 左右分割 / テキスト中央 | | max-width: ○○px |
| コンセプト | テキスト中央 / 左右分割（テキスト左＋画像右）| | |

| 施術事例 | 2列グリッド | | aspect-ratio: 1 |

### 画像の向きとレイアウトの対応

- **横長画像** → フルスクリーンヒーロー、ギャラリー行に向く
- **縦長画像** → 左右分割の片側に配置。横引き伸ばし絶対禁止
- **正方形画像** → グリッドカード、施術事例に向く

## 3. CSSパターン適用チェック（css-patterns-v5-final.mdから）

### カラー

- [ ] 背景: `#fff` → `#faf7f3`（ウォーム白）に置き換えたか
- [ ] テキスト色: 3段階以上のグレー階層を設定したか
- [ ] ボーダー: `#f1efe9`（ウォームボーダー）を使ったか
- [ ] セクション背景の濃淡: 交互に `--warm-white` / `--warm-white-deep`

### タイポグラフィ

- [ ] YakuHanMP CDNを読み込んだか
- [ ] 英語見出し: Josefin Sans w100 or Cormorant Garamond
- [ ] 日本語見出し: Noto Serif JP w300
- [ ] 本文: Zen Kaku Gothic New w300
- [ ] letter-spacing: 見出しは0.1em以上、本文は0.05〜0.06em
- [ ] line-height: セクションごとに変えたか（1.6〜2.2の範囲で最低3段階）

### 余白・レイアウト

- [ ] セクション間に十分な余白（padding: 80px〜120px）
- [ ] コンテンツ幅をmax-widthまたは%で制限したか
- [ ] 画像と画像の間に呼吸（gap、margin）があるか
- [ ] 密集していないか（スクショで俯瞰確認）

### インタラクション

- [ ] hover = opacity only（潔く）
- [ ] fade-in: Safari対策済みJS（DOMContentLoaded + function + フォールバック）
- [ ] cubic-bezier使ったか（ease-outだけで済ませてないか）

## 4. Safari/モバイル対策チェック

- [ ] base64埋め込みしていないか（画像は別ファイル参照）
- [ ] JSはアロー関数を使っていないか（function式に統一）
- [ ] DOMContentLoadedで包んでいるか
- [ ] IntersectionObserver非対応時のフォールバックがあるか
- [ ] observer.unobserveで表示済み要素を監視解除しているか
- [ ] -webkit-backdrop-filterを併記しているか
