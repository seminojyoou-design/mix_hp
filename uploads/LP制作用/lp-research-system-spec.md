# LPデザインリサーチ＆ストックシステム 設計書

## 概要

美容室（および他業種）のLP実例をギャラリーサイトから自動収集し、タグ付け分類してストックする仕組み。
テンプレート制作時の「参照データベース」として使う。

**目的：「デザインを生む」のではなく「デザインを選ぶ」体制を作る**

---

## 全体アーキテクチャ

```
STEP 1: 巡回・収集（Playwright MCP）
  ギャラリーサイトを巡回
  → LP個別ページのURL取得
  → 各LPのフルページスクショ撮影
  → メタ情報（タイトル、業種、カラー等）を抽出

STEP 2: 分析・タグ付け（Claude API）
  スクショをClaude Sonnetに渡す
  → 自動タグ付け（配色、レイアウト、雰囲気、構造パターン）
  → JSON形式で保存

STEP 3: ストック管理（スプレッドシート or JSON）
  全LPデータを一覧管理
  → 検索・フィルタリング可能に
  → テンプレ制作時に「こういう雰囲気で」と参照指定

STEP 4: テンプレ制作（ココ＝Claudeチャット）
  参照データのスクショ＋URLを受け取る
  → プロの実例を見ながらHTMLテンプレートを制作
  → Claude Codeはデータ差し替え担当（デザイン判断しない）
```

## STEP 1: 巡回・収集の詳細

### 巡回先リスト

#### Tier 1（LP専門ギャラリー・収穫量多い）
| サイト | URL | 特徴 |
|--------|-----|------|
| LP ARCHIVE | https://rdlp.jp/lp-archive | 国内最大級、カテゴリ検索あり |
| SANKOU! | https://sankoudesign.com | デザイン品質高め、業種タグあり |
| LP advance | https://lp-advance.com | LP特化、カラー検索可能 |
| Web Design Clip | https://webdesignclip.com | LP以外も含むがフィルタ可 |

#### Tier 2（参考になるが巡回は手動推奨）
| サイト | URL | 特徴 |
|--------|-----|------|
| Pinterest | https://pinterest.com | 「美容室 LP」で検索、量は多いが品質混在 |
| Dribbble | https://dribbble.com | 海外デザイン、モック多め |
| Awwwards | https://awwwards.com | 海外ハイエンド、美容室少なめ |

### 巡回スクリプト仕様（Claude Code + Playwright MCP用）

```
■ 入力
  - ギャラリーサイトURL
  - 検索カテゴリ: 「美容室」「ヘアサロン」「美容院」
  - 取得件数上限: 最大50件/サイト（初回）
  - 保存先ディレクトリ: ./lp-stock/screenshots/

■ 処理フロー
  1. ギャラリーサイトにアクセス
  2. カテゴリ or 検索で「美容室」に絞り込む
  3. 一覧ページからLP個別ページのURLを全取得
  4. 各LP個別ページにアクセス
     a. 実際のLPサイトURLを取得（ギャラリーからの外部リンク）
     b. LPサイトにアクセス
     c. viewport: 1440x900 でフルページスクショ撮影
     d. viewport: 375x812 でスマホ版スクショ撮影
     e. メタ情報を抽出:
        - サイトタイトル（<title>タグ）
        - メインカラー（背景色 or ヒーロー部分の支配色）
        - ギャラリー側のタグ情報（あれば）
  5. 保存:
     - スクショ: ./lp-stock/screenshots/{id}_pc.png, {id}_sp.png
     - メタ: ./lp-stock/meta/{id}.json

■ 出力（1件あたり）
  {
    "id": "beauty_001",
    "source": "rdlp.jp",
    "gallery_url": "https://rdlp.jp/lp-archive/xxxxx",
    "site_url": "https://example-salon.com",
    "title": "○○美容室",
    "category": "美容室",
    "screenshot_pc": "./screenshots/beauty_001_pc.png",
    "screenshot_sp": "./screenshots/beauty_001_sp.png",
    "collected_at": "2026-04-06T12:00:00"
  }

■ エラーハンドリング
  - LP先がリンク切れ → スキップしてログに記録
  - スクショがタイムアウト → 10秒待ってリトライ1回
  - CAPTCHAが出た → スキップしてログに記録
  - 同一ドメインの重複 → スキップ
```

### 注意事項
- ギャラリーサイトごとにDOM構造が違うので、サイト別にセレクタを調整する必要あり
- 初回はTier 1の1サイト（LP ARCHIVE推奨）だけで試して、動作確認してから横展開
- robots.txtを確認して、巡回禁止のサイトは対象外にする
- リクエスト間隔: 最低3秒空ける（サーバー負荷軽減）

## STEP 2: 分析・タグ付けの詳細

### Claude APIで自動タグ付け

スクショをbase64でClaude Sonnet APIに渡し、以下のタグを自動判定する。

```python
# タグ付けプロンプト（Claude Code内で実行）
prompt = """
この美容室LPのスクリーンショットを分析して、以下のJSON形式でタグ付けしてください。

{
  "color_scheme": {
    "primary": "メインカラー（例: ゴールド、ピンク、ネイビー）",
    "secondary": "サブカラー",
    "tone": "ライト / ダーク / ナチュラル / ビビッド"
  },
  "layout": {
    "hero_type": "フルスクリーン写真 / 左右分割 / テキスト中央 / スライダー / 動画",
    "menu_type": "カード型 / テーブル型 / リスト型 / アコーディオン",
    "overall": "1カラム / 2カラム / グリッド / マガジン"
  },
  "mood": "高級 / ナチュラル / ポップ / モード / 和風 / カジュアル / エレガント",
  "target": "20代女性 / 30代女性 / 男女 / ファミリー / メンズ",
  "features": [
    "ここが特徴的と思った要素を3つ（例: 縦書きテキスト、波形セクション区切り、パララックス）"
  ],
  "quality_score": 1-5の整数（デザインクオリティ主観評価）,
  "steal_point": "このLPから盗むべきテクニック1つ（例: ヒーロー画像のオーバーレイグラデ配分）"
}
"""
```

### タグ体系

```
■ 配色トーン
  ライト / ダーク / ナチュラル / ビビッド / パステル / モノクロ

■ ヒーロータイプ
  フルスクリーン写真 / 左右分割 / テキスト中央 / スライダー
  / 動画背景 / イラスト / アニメーション / なし（直接コンテンツ）

■ メニュー表示
  カード型 / テーブル型 / リスト型 / アコーディオン / タブ切替
  / 横スクロール / タイムライン / アイコン付き

■ レビュー表示
  カード並列 / 吹き出し / 引用ブロック / スライダー / 星付きリスト

■ CTA表示
  フル幅バナー / フローティングバー / カード型 / インライン

■ 雰囲気
  高級 / ナチュラル / ポップ / モード / 和風 / カジュアル
  / エレガント / インダストリアル / ミニマル / リゾート

■ 特殊演出
  パララックス / スクロールフェード / 縦書き / 波形区切り
  / マウスストーカー / ホバーズーム / テキストマスク / カウンター
```

## STEP 3: ストック管理

### ディレクトリ構造

```
lp-stock/
├── screenshots/
│   ├── beauty_001_pc.png
│   ├── beauty_001_sp.png
│   ├── beauty_002_pc.png
│   └── ...
├── meta/
│   ├── beauty_001.json  （STEP1のメタ + STEP2のタグ結合）
│   └── ...
├── catalog.json          （全件の一覧インデックス）
├── catalog.csv           （スプレッドシート用エクスポート）
└── viewer.html           （ローカルで見れるビューアー）
```

### catalog.json 構造

```json
[
  {
    "id": "beauty_001",
    "source": "rdlp.jp",
    "site_url": "https://example-salon.com",
    "title": "○○美容室",
    "category": "美容室",
    "screenshot_pc": "./screenshots/beauty_001_pc.png",
    "screenshot_sp": "./screenshots/beauty_001_sp.png",
    "color_scheme": { "primary": "ゴールド", "secondary": "ブラウン", "tone": "ダーク" },
    "layout": { "hero_type": "フルスクリーン写真", "menu_type": "カード型", "overall": "1カラム" },
    "mood": "高級",
    "target": "30代女性",
    "features": ["オーバーレイグラデ", "スクロールフェード", "セリフ体タイトル"],
    "quality_score": 4,
    "steal_point": "ヒーローのグラデが左右非対称で写真を殺さない",
    "status": "在庫",
    "used_for": null,
    "collected_at": "2026-04-06"
  }
]
```

### viewer.html（ローカルビューアー）

カタログをブラウザで一覧表示するシンプルなHTML。
- グリッド表示でスクショをサムネ一覧
- タグでフィルタリング（mood, color_scheme.tone, hero_type等）
- クリックで拡大表示 + メタ情報確認
- 「このデザイン使う」ボタン → statusを「使用済み」に変更

**これもClaude Codeに作らせる（React or バニラHTML）**

## STEP 4: テンプレ制作フロー（改訂版）

```
1. viewerで参照LPを2〜3個選ぶ
   「A01のヒーロー ＋ B03のメニュー構成 ＋ C07の配色」みたいに

2. 選んだスクショとsteal_pointをココ（Claudeチャット）に貼る
   「このヒーローのグラデ配分と、このメニューの横スクロールを
    組み合わせて、配色はターコイズ系でテンプレ作って」

3. ウチがHTMLテンプレートを制作

4. テンプレをlp-stockのtemplates/に追加
   Claude Codeはこのテンプレのデータだけ差し替える係

5. 使った参照LPのstatusを「参照済み」に変更
   次のテンプレ制作時にミックスの種として使える
```

## 重要な設計思想

1. **デザインは「生む」のではなく「選ぶ」**
   → 0からの発想ではなく、プロの実例から構造を学んで組み合わせる

2. **Claude Codeにデザイン判断させない**
   → テンプレートのデータ差し替え担当。デザインはココ（Claudeチャット）で作る

3. **在庫管理型**
   → テンプレは使ったら「使用済み」に。参照データは「参照済み」に。常に鮮度を保つ

4. **アファンタジア対応**
   → 視覚イメージではなく「構造タグ × 参照スクショ」の組み合わせで指定する
